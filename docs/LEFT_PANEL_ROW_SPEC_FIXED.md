# LEFT PANEL ROW SPEC FIXED

목표:
좌측 실거래 관리종목 리스트의 컬럼 의미를 아래 기준으로 고정한다.
이번 내용은 다음 Cursor 패치의 기준 스펙이다.
해석하지 말고 그대로 반영 기준으로 사용한다.

==================================================
1. 컬럼 최종 정의
==================================================

최종 컬럼 순서:

1) 순위
2) 구분
3) 종목
4) AI 점수
5) 상태
6) 비중
7) 목표비중
8) 액션

==================================================
2. 각 컬럼 의미
==================================================

--------------------------------------------------
2-1. 순위
--------------------------------------------------

의미:
비중 순위

정렬 기준:
- 보유 중인 종목은 "보유 비중"이 높은 순
- 보유하지 않고 대기 중인 종목이 여러 개라면 "AI 점수"가 높은 순

즉 우선순위 규칙:
1. 보유 종목 우선
2. 보유 종목끼리는 보유 비중 내림차순
3. 미보유 대기 종목끼리는 AI 점수 내림차순

중요:
이건 표시 순서 기준이다.

--------------------------------------------------
2-2. 구분
--------------------------------------------------

의미:
이 종목을 관리종목에 넣은 주체가 누구인가

표기:
- [B] = 기본 스테이블 3총사
- [A] = AI 픽
- [U] = 유저 픽

색상:
- B = 녹색 계열
- A = 파랑 계열
- U = 회색 계열

형태:
작은 원형 또는 pill 배지 1개

중요:
기존 코인 로고 대체
코인 로고 자동 불가하면 이 배지 방식으로 고정

--------------------------------------------------
2-3. 종목
--------------------------------------------------

표시 방식:
1행 = 영어 약자
2행 = 한글 코인 이름

예:
BTC
비트코인

중요:
목록에서 가져온 종목 중 한글 이름이 안 나오는 경우가 있으므로
현재 데이터 source에서 한글명 매핑이 누락되는 종목 체크 필요

반영 원칙:
- 한글명 있으면 2행에 표시
- 없으면 최소한 영어 약자만 정상 표시
- 빈칸으로 두지 말 것

--------------------------------------------------
2-4. AI 점수
--------------------------------------------------

의미:
AI가 Upbit API 기반으로 계산한 점수

주의:
이미 구현되어 있는 기존 점수 로직을 그대로 사용
새 계산 로직 만들지 말 것

표시 방식:
- 숫자
- 작은 score bar

--------------------------------------------------
2-5. 상태
--------------------------------------------------

의미:
지금 AI가 해당 종목에 대해 무엇을 하고 있는가

예시:
- 관망
- 매수대기
- 매도중
- 매수검토
- 매매보류

중요:
이 열은 "현재 AI의 행동 상태"를 보여주는 열이다.
이전처럼 부가 데이터(% / ETA)를 과하게 섞지 말고
핵심 상태 중심으로 표시

--------------------------------------------------
2-6. 비중
--------------------------------------------------

의미:
현재 보유 중인 코인의 실제 보유 비중

예:
35%

미보유면:
- 0%
- 또는 —
현재 시스템 기준에 맞추되 일관되게 표시

--------------------------------------------------
2-7. 목표비중
--------------------------------------------------

의미:
AI 또는 사용자 설정 기준으로 목표하는 비중

예:
현재 비중 35%
목표비중 50%
→ 사용자는 아직 추가 매수가 가능하다고 인지

중요:
이 열은 새 기능 성격이므로 반드시 확인 필요

확인 사항:
- 현재 코드에 목표비중 데이터 source가 이미 있는지
- 없으면 UI placeholder만 먼저 둘지
- 사용자가 수정 가능하게 만들 수 있는 기존 경로가 있는지

원칙:
이번 다음 패치에서는
1. 목표비중 데이터 존재 여부부터 확인
2. 있으면 표시
3. 없으면 임의값 넣지 말 것
4. 수정 가능 기능은 "현재 코드에 기반이 있는 경우만" 연결

--------------------------------------------------
2-8. 액션
--------------------------------------------------

의미:
행별 관리 액션

최소 항목:
- 삭제
- 필요 시 보류

중요:
삭제는 관리종목에서만 제거
Explorer/후보 목록에는 남아 있어야 함

==================================================
3. 헤더 영역 최종 정의
==================================================

상단 1행:
AI MANAGED CANDIDATES [실거래 관리종목]

상단 2행:
관리 n종목 | 보유 n | 후보 n

버튼:
- 매매 보류
- - 삭제

중요:
+추가 버튼 삭제
정렬 버튼 삭제
더보기 버튼 삭제

==================================================
4. 리스트 UI 방향
==================================================

리스트는 단순 테이블 느낌이 아니라
운영 리스트 / 카드형 리스트처럼 보여야 한다.

원칙:
- 각 행은 독립된 운영 카드 느낌
- 선보다 spacing과 정렬이 중요
- 열 너비는 의미 중심으로 재조정
- 코인 로고 대신 B/A/U 배지 사용

==================================================
5. 목표비중 관련 확인 과제
==================================================

다음 Cursor 패치 전 확인 필요:

1. 현재 코드에 목표비중 source가 있는가
2. 사용자 수정 기능이 이미 일부라도 존재하는가
3. 없다면 이번엔 표시만 할지, 다음 단계로 넘길지

중요:
목표비중은 새 기능이므로
확인 없이 임의 구현 금지

==================================================
6. 다음 패치 범위
==================================================

다음 Cursor 패치는 아래만 처리하면 된다.

1. 헤더 버튼 정리 (+추가 삭제)
2. 리스트 컬럼 재정의
3. 구분 배지 B/A/U 적용
4. 종목 2줄 표시 안정화
5. 상태 열 의미 정리
6. 목표비중 source 확인 후 표시 여부 결정
